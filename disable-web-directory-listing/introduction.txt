After, I listed all the files in my directory with ls -la to list the hidden file as well, and changed the server environment variable to disable directory listing to false. 
The, I ran the restart server script to apply the changes. 
Finally, to see the changes, I ran curl -i http://120.0.0.1:18280/downloads/ and the directory listing is diabled.

Summary
In this challenge, I handled a realistic web hardening cleanup after a content rollout. I disable a browsable directory listing, restarted the service through the existing helper script, and verified that legitimate file delivery still worked. This is a practical DevSecOps task: reduce unnecessary information exposre without breaking normal content access.
The site remains useful to visitors, but the directory structure is no longer exposed an an index.
